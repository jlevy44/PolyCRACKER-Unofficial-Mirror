Welcome to polyCRACKER’s documentation!¶

python polyCRACKER.py¶

python polyCRACKER.py [OPTIONS] COMMAND [ARGS]...

Options

--version¶

Show the version and exit.

TE_Cluster_Analysis¶

Build clustering matrix (repeat counts vs scaffolds) from repeats and conduct analyses on repeats instead of kmers.

python polyCRACKER.py TE_Cluster_Analysis [OPTIONS]

Options

-w, --work_dir <work_dir>¶

Work directory where computations for TE clustering is done.

-rg, --repeat_gff <repeat_gff>¶

Full path to repeat TE gff file, output from repeat masker pipeline.

-rp, --repeat_pickle <repeat_pickle>¶

Full path to repeat TE pickle file, containing TE names, which will be generated in this pipeline.

-sl, --split_length <split_length>¶

Length of intervals in bedgraph files. [default: 75000]

-n, --n_chromosomes <n_chromosomes>¶

Output unionbed plots of x largest chromosomes. [default: 10]

-s, --n_clusters <n_clusters>¶

Number of clusters for clustering TEs. [default: 2]

-d, --n_dimensions <n_dimensions>¶

Number of dimensions to reduce to. [default: 3]

-k, --kernel <kernel>¶

Kernel for KPCA. Cosine is particularly effective. [default: cosine]

-m, --blast_mem <blast_mem>¶

Amount of memory to use for bbtools run. More memory will speed up processing. [default: 100]

-nt, --n_top_repeats <n_top_repeats>¶

Number of top TEs in each subgenome to analyze. [default: 100]

-fai, --fai_file <fai_file>¶

Full path to original, nonchunked fai file.

-dv, --default_repeat_val <default_repeat_val>¶

Default value for number of times TE appears in subgenome if value is 0. [default: 2]

-dt, --diff_repeat_threshold <diff_repeat_threshold>¶

Threshold specifying that if one TE is differential in one subgenome versus others label that TE by that subgenome. [default: 20]

-bo, --original_subgenomes_bed <original_subgenomes_bed>¶

Full path to original bed file containing subgenome labelling. Will use this to label TEs by subgenome instead of clustering. Leave blank if deciding labels via clustering.

bio_hyp_class¶

Generate count matrix of kmers versus genomes. Optionally can transpose this matrix and find kmer guide tree / approximated phylogeny based on differential kmer counts. Changing settings to include all kmers can generate a more accurate guide tree.

python polyCRACKER.py bio_hyp_class [OPTIONS]

Options

-i, --input_dict <input_dict>¶

MANDATORY: Comma delimited mapping of all subgenome names to their subgenome fasta files. [default: subgenome_1:./fasta_files/subgenome_1.fa,SpeciesA:./SpeciesA.fasta]

-f, --folder_delimiter <folder_delimiter>¶

OPTIONA: Comma delimited list of folder containing genomes, delimiters from which to construct short names, and number of characters to use for short name. Save fastas as AAAA_xxx_v0.fa format. Supercedes -i option, unless default value not modified. [default: ./folder/,_,4]

-kv, --default_kcount_value <default_kcount_value>¶

If kmer hits in subgenome is 0, this value will replace it. [default: 1]

-dk, --ratio_threshold <ratio_threshold>¶

Value indicates that if the counts of a particular kmer in a subgenome divided by any of the counts in other subgenomes are greater than this value, then the kmer is differential. [default: 20]

-w, --work_dir <work_dir>¶

Work directory where computations for biological hypothesis testing is done. [default: ./]

-m, --blast_mem <blast_mem>¶

Amount of memory to use for bbtools run. More memory will speed up processing. [default: 100]

-l, --kmer_length <kmer_length>¶

Length of kmers to find. [default: 23]

-ft, --fast_tree¶

Make quick estimated phylogenetic tree out of kmer matrix.

-min, --min_count <min_count>¶

Minimum count for kmer counting. [default: 3]

-sk, --skip_kcount¶

Skip generation of kcount files. kcount files must be named by the following convention: subgenome_1 used in input dict -> subgenome_1.kcount

-max, --max_val <max_val>¶

Minimum count for maximum count of kmer between subgenomes. [default: 100]

cluster¶

Perform clustering on the dimensionality reduced data, though various methods. Takes in npy position file and outputs clustered regions and plot.

python polyCRACKER.py cluster [OPTIONS]

Options

-f, --file <file>¶

Numpy .npy file containing positions after dimensionality reduction

-c, --cluster_method <cluster_method>¶

Clustering method to use. [default: SpectralClustering]

-s, --n_subgenomes <n_subgenomes>¶

Number of subgenomes [default: 2]

-m, --metric <metric>¶

Distance metric used to compute affinity matrix, used to find nearest neighbors graph for spectral clustering. [default: cosine]

-nn, --n_neighbors <n_neighbors>¶

Number of nearest neighbors in generation of nearest neighbor graph. [default: 10]

-wnn, --weighted_nn <weighted_nn>¶

Whether to weight spectral graph for spectral clustering. [default: 0]

-all, --grab_all <grab_all>¶

Whether to grab all clusters, and number of clusters would equal number of subgenomes. By default, the number of clusters is one greater than the number of subgenomes, and the algorithm attempts to get rid of the ambiguous cluster [default: 0]

-g, --gamma <gamma>¶

Gamma hyperparameter for Spectral Clustering and BGMM models. [default: 1.0]

-mst, --min_span_tree <min_span_tree>¶

Augment k nearest neighbors graph with minimum spanning tree. [default: 0]

dash_genome_quality_assessment¶

Input pre chromosome level scaffolded genomes, before split by ploidy, and outputs frequency distribution of select kmers. Used to compare this distribution of kmers to that found from comparing post-scaffolded polyploid subgenomes. The scaffolding, if reference based, may be biased and missing key repeat information, so comparing to prescaffolded genomes may show whether repeat info was lost during scaffolding for important words.

python polyCRACKER.py dash_genome_quality_assessment [OPTIONS]

Options

-w, --work_dir <work_dir>¶

Work directory where computations for finding array of kmers versus prescaffolded genomes with unassigned sequences. [default: ./]

-csv, --kmer_rule_matrix <kmer_rule_matrix>¶

Kmer count matrix with appended rule labels. [default: ./kmer_master_matrix_rules.csv]

-g, --pre_scaffolded_genomes <pre_scaffolded_genomes>¶

Amount of memory to use for bbtools run. More memory will speed up processing. [default: 100]

diff_kmer_analysis¶

Runs robust differential kmer analysis pipeline. subgenomeExtraction¶

Extract subgenomes from genome, either chunked genome or original genome can be fed in as argument

python polyCRACKER.py subgenomeExtraction [OPTIONS]

Options

-s, --subgenome_folder <subgenome_folder>¶

Subgenome folder where subgenome extraction is taking place. This should be the bootstrap_0 folder.

-os, --original_subgenome_path <original_subgenome_path>¶

One level up from subgenomeFolder. Final outputs are stored here.

-p, --fasta_path <fasta_path>¶

Directory containing chunked polyploids fasta file [default: ./fasta_files/]

-g, --genome_name <genome_name>¶

Filename of chunked polyploid fasta.

-go, --original_genome <original_genome>¶

Filename of polyploid fasta; can be either original or chunked, just make sure original is set to 1 if using original genome.

-bb, --bb <bb>¶

Whether bbtools were used in generating blasted sam file. [default: 1]

-b, --bootstrap <bootstrap>¶

Number of times to bootstrap the subgenome extraction process. Each time you bootstrap, you should get better classification results, else instability has occurred due to ambiguous clustering. [default: 0]

-i, --iteration <iteration>¶

Current iteration of bootstrapping process. Please set to 0, when initially beginning. [default: 0]

-l, --kmer_length <kmer_length>¶

Length of kmers to find; can include multiple lengths if comma delimited (e.g. 23,25,27) [default: 23,31]

-r, --run_final <run_final>¶

Turn this to 0 when starting the command. Script terminates if this is set to 1. [default: 0]

-o, --original <original>¶

Select 1 if trying to extract subgenomes based on original nonchunked genome. [default: 0]

-m, --blast_mem <blast_mem>¶

Amount of memory to use for bbtools run. More memory will speed up processing. [default: 100]

-kl, --kmer_low_count <kmer_low_count>¶

Omit kmers from analysis that have less than x occurrences throughout genome. [default: 100]

-dk, --diff_kmer_threshold <diff_kmer_threshold>¶

Value indicates that if the counts of a particular kmer in a subgenome divided by any of the counts in other subgenomes are greater than this value, then the kmer is differential. [default: 20]

-u, --unionbed_threshold <unionbed_threshold>¶

Two comma delimited values used for binning regions into subgenomes. First value indicates that if the counts of all kmers in a particular region in any other subgenomes are 0, then the total count in a subgenome must be more than this much be binned as that subgenome. Second value indicates that if any of the other subgenome kmer counts for all kmers in a region are greater than 0, then the counts on this subgenome divided by any of the other counts must surpass a value to be binned as that subgenome. [default: 10,2]

-ds, --diff_sample_rate <diff_sample_rate>¶

If this option, x, is set greater than one, differential kmers are sampled at lower frequency, and total number of differential kmers included in the analysis is reduced to (total #)/(diff_sample_rate), after threshold filtering [default: 1]

-kv, --default_kmercount_value <default_kmercount_value>¶

If a particular kmer is not found in a kmercount dictionary, this number, x, will be used in its place. Useful for calculating differential kmers. [default: 3.0]
